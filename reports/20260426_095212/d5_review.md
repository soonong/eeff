## 핵심 이슈 및 원인

### 이슈 1: 공고명/발주처/공고번호 전 케이스 null

**원인**: 나라장터 공고문 PDF 구조에서 공고명과 발주처는 주로 공고문 상단 헤더에 표시되나, pypdf의 텍스트 추출에서 손실됨. PDF에서 앞 5페이지를 추출해도 구조적으로 파싱이 안 되는 것으로 판단.

**해결 방안**:
- pdfplumber 전환 (layout-aware 추출, 더 높은 정확도)
- 또는 bidding2 API 응답의 메타데이터에서 공고명 추출
- 또는 나라장터 공고 API 직접 연동 (공고번호로 공고명/발주처 조회)

### 이슈 2: bidding2 8/13건 응답 없음

**원인**: 공고번호 형식 차이. `R26BK*` 형식만 응답 — `E08260*`, `G01260*` 형식은 다른 시스템(한전, 지자체) 공고번호.

**해결 방안**: 공고번호 prefix별 라우팅 — R26BK는 bidding2, E08*는 kepco, G01*는 별도 처리.

### 이슈 3: 모델 변경 필요

**원인**: `gemini-1.5-flash-002` Context Caching 미지원, `gemini-2.0-flash` free tier 쿼터 소진.  
**현재 적용**: `gemini-flash-latest` (=gemini-3-flash 계열, 1회 호출 20~240초).
