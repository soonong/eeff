# 참가자격(T2) 추출 예시

`참가자격` 필드는 공고문 내 **참가자격 전문(원문 발췌)**.
원문 그대로 추출하되, 관련 없는 공고 전문(일정·금액 등)은 포함하지 않는다.

## 주의

> **잘못 추출하기 쉬운 패턴 (negative 1 참고):**
> - 공고문 전체를 이 필드에 삽입하는 오류.
> - 참가자격 항목이 여러 곳에 분산되어 있을 때 일부만 발췌하거나, 반대로 관련 없는 단락까지 포함하는 오류.
> - "가. 나. 다." 항목 중 참가자격 내용이 아닌 안전관리계획서 제출 의무 같은 의무사항을 포함하는 오류.

---

## Positive 1 — 표준 원문 추출

공고문 원문:
```
4. 참가자격
   가. 건설업(토목공사업) 등록 업체
   나. 지역업체(서울특별시) 의무 참여 49% 이상
   다. 부정당업자 제재 중인 업체 참여 불가
```

출력:
```json
{
  "참가자격": "가. 건설업(토목공사업) 등록 업체\n나. 지역업체(서울특별시) 의무 참여 49% 이상\n다. 부정당업자 제재 중인 업체 참여 불가"
}
```

---

## Positive 2 — 동의어·형식 차이

공고문 원문:
```
[입찰참가자격]
○ 수도공사업 면허 보유자
○ 신용평가 B 이상
※ 지역 제한 없음
```

출력:
```json
{
  "참가자격": "○ 수도공사업 면허 보유자\n○ 신용평가 B 이상\n※ 지역 제한 없음"
}
```

규칙: "[입찰참가자격]" 같은 섹션 헤더는 제외하고 내용만 추출.

---

## Negative 1 — 잘못 추출하기 쉬운 패턴

공고문 원문:
```
1. 공사명: OO도로 포장공사
2. 입찰일시: 2026-05-23 11:00
3. 참가자격: 포장공사업 면허 보유자
4. 안전관리: 안전관리계획서 제출 의무
5. 기타: 현장설명회 필참
```

**잘못된 출력** (관련 없는 항목 포함):
```json
{
  "참가자격": "포장공사업 면허 보유자\n안전관리계획서 제출 의무\n현장설명회 필참"
}
```

**올바른 출력:**
```json
{
  "참가자격": "포장공사업 면허 보유자"
}
```

이유: 안전관리·현장설명회는 참가자격이 아니라 별도 의무사항이다. 3번 항목만 추출한다.
