# 특수실적(T2) 추출 예시

`특수실적` 필드는 공고문에 명시된 특수 시공 실적 요건 텍스트를 **원문 그대로** 추출한다.
자유 텍스트이므로 AI가 임의 해석·요약·생략하지 말 것. 없으면 null.

## 주의

> **잘못 추출하기 쉬운 패턴 (negative 1 참고):**
> - `참가자격` 전문에 포함된 일반 자격 요건을 `특수실적`으로 오인하는 경우.
> - "해당 없음", "없음" 같은 명시적 표기를 빈 텍스트가 아니라 null로 처리해야 한다.

---

## Positive 1 — 표준 원문 추출

공고문 원문:
```
특수공사 시공실적 기준:
최근 5년간 아스팔트 포장공사 완공 실적 합계 금액이 15억 원 이상인 자
```

출력:
```json
{
  "특수실적": "최근 5년간 아스팔트 포장공사 완공 실적 합계 금액이 15억 원 이상인 자"
}
```

규칙: 머리글("특수공사 시공실적 기준:") 같은 레이블 없이 실질 요건 텍스트만 추출.

---

## Positive 2 — 동의어·형식 차이

공고문 원문:
```
특수실적요건) 교량공사 시공경험이 있는 자로서 연간 계약금액이 10억 원 이상인 건을 1건 이상 준공한 자
```

출력:
```json
{
  "특수실적": "교량공사 시공경험이 있는 자로서 연간 계약금액이 10억 원 이상인 건을 1건 이상 준공한 자"
}
```

규칙: "특수실적요건)" 같은 변형 헤더를 걷어내고 내용만 추출.

---

## Negative 1 — 잘못 추출하기 쉬운 패턴

공고문 원문:
```
참가자격:
  가. 건설업 등록 업체
  나. 지역 업체(서울) 49% 이상 참여 의무
  ※ 별도 시공 실적 요건 없음
```

**잘못된 출력** (오인 사례):
```json
{
  "특수실적": "지역 업체(서울) 49% 이상 참여 의무"
}
```

**올바른 출력:**
```json
{
  "특수실적": null
}
```

이유: "별도 시공 실적 요건 없음" → 특수실적 없음. 지역 의무 비율은 `지역의무비율` 필드 대상.
