# 특수실적_내용(T2) 추출 예시

`특수실적_내용` 필드는 공고문에서 **선택적 실적(택1) 내용** 원문.
"아래 중 하나 충족", "다음 중 하나 이상", "택1" 등의 표현이 수반되는 복수 옵션 실적 요건을 그대로 추출.

## 주의

> **잘못 추출하기 쉬운 패턴 (negative 1 참고):**
> - 공통 필수 요건(모두 충족해야 함)을 선택적 내용으로 오인하는 경우.
> - "해당 없음" 표기가 있는 경우 빈 텍스트("")가 아닌 null로 처리해야 한다.
> - 수치·금액만 발췌하고 조건 문맥을 잘라내는 오류.

---

## Positive 1 — 표준 원문 추출

공고문 원문:
```
다음 중 하나 이상 충족:
  ① 최근 5년간 교량 상부 공사 완공 실적 30억 원 이상
  ② 최근 5년간 터널 공사 완공 실적 20억 원 이상
```

출력:
```json
{
  "특수실적_내용": "① 최근 5년간 교량 상부 공사 완공 실적 30억 원 이상\n② 최근 5년간 터널 공사 완공 실적 20억 원 이상"
}
```

---

## Positive 2 — 동의어·형식 차이

공고문 원문:
```
[선택실적] 아래 항목 중 택1 충족
- 하수관거 정비공사 준공 10억 이상(최근 3년)
- 상수도 시설공사 준공 8억 이상(최근 3년)
```

출력:
```json
{
  "특수실적_내용": "하수관거 정비공사 준공 10억 이상(최근 3년) / 상수도 시설공사 준공 8억 이상(최근 3년)"
}
```

---

## Negative 1 — 잘못 추출하기 쉬운 패턴

공고문 원문:
```
특수실적 요건:
  모든 구성원: 건설업 등록 업체일 것
  ※ 별도 선택 실적 요건 해당 없음
```

**잘못된 출력** (공통 요건을 내용으로 오인 또는 "해당 없음"을 빈 텍스트로 처리):
```json
{
  "특수실적_내용": ""
}
```

**올바른 출력:**
```json
{
  "특수실적_내용": null
}
```

이유: "해당 없음" 명시 → null. 빈 문자열("")이 아닌 null이 정확하다.
